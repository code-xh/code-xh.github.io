本篇博客是 Windows10 平台下基于 hexo 搭建静态博客的攻略以及经验分享，有一些我自己踩过的坑及解决办法。

在此特别感谢 B站up CodeSheep 所提供的参考帮助！

仅以此篇开始我的个人博客之旅，同时也希望能够对初用 hexo 的大家有所帮助。

软件的安装与准备

1、Git 的安装

Git是用来管理代码的，在 Windows 下用 cmd 貌似也可以实现，但比较麻烦，所以推荐下载此软件。

Git官网

缺省安装即可（即一路 next）。
安装成功后，随意在一个文件夹中右键，会出现 Git Bash Here

2、Node.js 的安装

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。使用 hexo 需要安装此环境。

Nodejs官网

选择左边的 LTS 版本，该版本更稳定。缺省安装即可。

Hexo 的安装

1、在随意一个文件夹的空白处右键鼠标，选择

2、输入 node -v 查看 Node.js 是否正确安装。例如，我此处显示 v12.18.2 。

3、更改 npm 的默认源。npm 是 Node.js 内置的可用于下载资源的软件包管理工具，它默认的源在国外，所以国内用户下载速度可能会有点慢。
所以此处我们给 npm 换一个源（此处我们是换成淘宝的）：
输入 npm install -g cnpm --registry=http://registry.npm.taobao.org

因为我已经安装过了，所以我这执行这些命令它是更新，会和大家第一次安装显示的内容有所不同。
安装完成后，输入 cnpm -v 可查看 cnpm 版本。

出现这样的内容，则说明 cnpm 安装成功。

4、输入 cnpm install -g hexo-cli 安装 hexo 框架

5、输入 hexo -v 查看 hexo 版本，验证 hexo 是否安装成功。

新建 Hexo 博客项目

1、新建一个文件夹，用于部署新建的博客项目。例如我在 E 盘中新建了一个名为 Blog1 的空文件夹。

2、双击进入此新建的文件夹，右键鼠标，选择
*注：与此项目有关的 git 终端都需要在这个目录下输入相关命令。可以选择打开次文件夹，然后右键选择 Git Bash Here；或者在 git 终端中使用 cd 你的文件夹地址 命令切换到该部署项目的目录。

3、键入命令 hexo init 部署项目

此时新建的文件夹中会出现部署产生的文件。Hexo 博客项目就新建好了。

运行测试

1、在 git 终端键入命令 hexo s 启动本地博客，用于预览博客，以后写的博客内容可以先本地预览后再上载。

打开浏览器，输入 localhost:4000 ，看到下图则说明本地博客搭建完成。
